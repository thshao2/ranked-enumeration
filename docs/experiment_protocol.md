# Experiment Protocol

## Goals
- Validate correctness against brute-force materialize+sort oracle.
- Measure preprocessing latency, first-result latency, per-result delay, and throughput.
- Compare ranked iterator against full materialization baseline.

## Commands
```bash
python3 -m pytest -q
python3 bench/bench_topk.py --shape path --size 4 --domain 20 --p 0.08 --k 50 --seed 0
python3 bench/bench_delay.py --shape path --size 4 --domain 20 --p 0.08 --limit 200 --seed 0
python3 bench/bench_compare_materialize_sort.py --shape path --size 4 --domain 20 --p 0.08 --seed 0
```

Repeat for star workloads:
```bash
python3 bench/bench_topk.py --shape star --size 4 --domain 20 --p 0.08 --k 50 --seed 0
python3 bench/bench_delay.py --shape star --size 4 --domain 20 --p 0.08 --limit 200 --seed 0
python3 bench/bench_compare_materialize_sort.py --shape star --size 4 --domain 20 --p 0.08 --seed 0
```

## Reporting Template
For each workload configuration, record:
- `shape`, `size`, `domain`, `p`, `seed`
- preprocessing seconds
- top-k time / full enumeration time
- first-result latency
- mean delay, p95 delay
- throughput rows/sec
- baseline materialize+sort time
- whether outputs match baseline exactly

## Notes
- Random workloads are generated by independent Bernoulli sampling over relation tuples.
- Use multiple seeds and aggregate mean/std for a course report.
