# Ranked Enumeration Design (Acyclic CQs, v1)

## Scope
- Acyclic conjunctive queries.
- User-provided tree decomposition.
- Full output variables only (no projection).
- Set semantics and deterministic tie-breaking `(score, output_tuple_lex)`.

## Pipeline
1. Validate query/relations/decomposition.
2. Assign each atom to the highest decomposition bag that contains its variables.
3. Build local bag relations by joining owned atoms.
4. Run one bottom-up and one top-down semijoin reducer pass.
5. Build per-key lazy ranked streams (`Q_t[u]`) for each node.
6. Enumerate from root stream in ranked order.

## Data Structures
- `BagRelation`: tuples over a bag variable ordering.
- `NodeState`: per decomposition node context, grouped local tuples by separator key.
- `KeyStream`: lazy min-heap frontier over `(theta, child_indices)` states.
- `Cell`: selected local tuple + child choices + score + cached successor + merged assignment.

## Enumeration Logic
- Root answers come from successive cells in `Q_root[()]`.
- Successors are generated by incrementing one child index at a time for a popped state.
- Child cells are fetched lazily and memoized inside child streams.

## Determinism
- Heap ordering uses `(score, output_tuple, theta, child_indices, serial)`.
- Public result order is stable across runs and repeated iterations.
